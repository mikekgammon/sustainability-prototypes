Foundation Phase · Requirement 04

Project Upload Template

Flowchart mapping the bulk project upload workflow — template structure, validation logic, upload processing, the edit/update cycle for existing projects, and the AI proposal scraping feature. Currently the most mature item in Phase 1 (In Progress).

Problem: The sustainability team currently creates projects one at a time in the platform. For large client accounts with dozens of energy measures, this is slow and error-prone. A standardised upload template lets the team prepare project data offline (or scrape it from proposals), then batch-upload it into TrueDI in one go.

1. Template Structure & Field Definitions
Consultant needs to upload projects in bulk
Download
Download upload template from platform
Standardised spreadsheet template (.xlsx) available from the project management area. Pre-formatted with column headers matching the TrueDI project data model. Includes a reference sheet with valid values for dropdowns (project types, fuel types, units, technology categories).
Template fields
Template column structure
Each row = one project. Columns map to the TrueDI project data model. Required fields must be populated for upload to succeed; optional fields enhance reporting.

Populate
Consultant fills in the template
Can be done manually from audit reports, proposals, or client data. For large datasets, the AI scraping feature (Flow 4) can auto-populate the template from proposal documents. Template supports multiple projects per sheet — one row per project.
2. Upload & Validation Pipeline
Consultant uploads completed template
Parse
Platform reads and parses the file
Accepts .xlsx format. Reads column headers and maps to internal data model. Counts rows to determine number of projects. Ignores blank rows and the reference sheet.
Validation
Row-level validation
Each row checked against validation rules: required fields populated? Values within expected ranges? Project type and unit valid against reference list? Numbers parseable? No duplicate project names?
All rows valid
Preview
Show upload preview
Summary screen: X projects to be created, auto-calculated carbon savings, total investment, breakdown by project type. Consultant reviews before confirming.
Errors found
Error report
Validation error summary
List of rows with errors: row number, field name, issue description (e.g. "Row 14: Unit — 'gallons' not in valid list"). Consultant fixes in the spreadsheet and re-uploads. Partial uploads not allowed — all or nothing.
On successful validation
Confirm
Consultant confirms upload
Single confirm action creates all projects in the client account. Each project gets full data model record including emission factor auto-lookup (if carbon saving not provided, platform calculates from consumption and unit). Projects immediately available in comparisons and strategies.
Result
Upload confirmation report
Summary: X projects created successfully. Link to each project. Any auto-calculated fields highlighted for review (e.g. "carbon saving derived from DEFRA factor — verify"). Upload logged in audit trail with consultant identity and timestamp.
3. Editing Existing Projects via Template
Savings updated — need to edit projects in bulk
Export
Export existing projects as template
Consultant downloads current projects for a client account as a pre-filled template. Same format as the upload template but populated with live data. Includes a project_id column (hidden/locked) so the platform can match edits to existing records.
Edit
Consultant edits values in spreadsheet
Update savings figures, costs, consumption values, or any other field. Can also add new rows for new projects (blank project_id = new project). Can remove rows to leave those projects untouched — only rows present in the upload are processed.
Re-upload
Upload edited template
Same validation pipeline as Flow 2. Platform detects project_id column and switches to update mode.
Decision
New or existing project?
For each row, platform checks whether a project_id is present.
Has project_id
Update
Update existing project
Platform diffs the uploaded values against the current record. Shows a change summary: "Saving changed from 45,000 to 52,000 kWh" for each modified field. Consultant confirms. Previous values retained in version history.
No project_id
Create
Create new project
Treated as a new upload. Same creation logic as Flow 2. Allows mixed uploads — update some projects, create others, all in one template.
Both processed together
Result
Update confirmation report
Summary: X projects updated, Y projects created, Z fields changed. Change log for audit trail. Carbon and cost totals recalculated for the account. Strategies and comparisons that reference updated projects are flagged for review.
4. AI Proposal Scraping (Future Enhancement)
Consultant has a proposal document (PDF/Word)
AI — Upload
Upload proposal document
Consultant uploads an energy audit report, investment proposal, or client tender response. Could be PDF, Word, or Excel. The platform ingests the document for AI extraction.
AI — Extract
AI extracts project data from document
Natural language processing identifies energy measures, technologies, savings figures, costs, payback periods, and site information from the proposal text and tables. Maps extracted values to the upload template fields. Handles common proposal formats (e.g. ESOS audit reports, investment grade proposals).
AI — Populate
Auto-populate upload template
AI-extracted data fills the standard template. Each field marked with a confidence indicator: high (exact match from document), medium (inferred from context), low (best guess — needs review). Consultant reviews and corrects before uploading.
Review
Consultant reviews AI output
Pre-upload review screen showing the extracted data alongside the source document. Low-confidence fields highlighted for attention. Consultant can edit any field, accept or reject individual rows, or discard and start again.
Accepted
Upload
Proceed to standard upload
Template enters the same validation pipeline as Flow 2. AI-populated data treated identically to manually-entered data from this point forward.
Needs corrections
Edit
Download as editable template
Export the AI-populated template as .xlsx for offline editing. Consultant corrects values, adds missing data, then uploads manually via Flow 2.
5. Data Integrity & Audit Trail
Integrity
Upload audit logging
Every upload (create or update) is logged with: consultant identity, timestamp, client account, number of projects affected, source (manual template or AI-scraped), and a snapshot of the uploaded data. Enables rollback if needed.
Versioning
Project version history
Each project maintains a version history. When updated via template re-upload, previous values are preserved. Consultant or client can view change log: "Saving changed from 45,000 to 52,000 kWh on 16 Apr 2026 by Otto Terrell via template upload".
Downstream
Impact propagation
When projects are updated, the platform recalculates affected comparisons and strategies. If a saving figure changes, the strategy's total carbon and cost projections update automatically. Users of affected strategies receive a notification that source data has changed.
