Project Data

Scope 3 Foundation

Today the platform models projects almost exclusively in kWh — it works for electricity and gas, but falls apart for anything outside Scope 2. The Scope 3 Foundation expands this so a project can represent any type of intervention (fuel switching, supply chain changes, waste reduction) measured in any unit (litres, tonnes, kilometres, not just kWh).

What we need to know about each project

Type of project GHG Scope (1, 2, or 3) What's being measured Unit of measurement Baseline consumption Projected consumption Emission factor Cost before & after What drives the value

Project types the platform will support

Energy efficiency Fuel switching Supply chain Process change Other

Units of measurement (beyond kWh)

kWh Litres Tonnes Kilometres m³ kg Custom
Key design principle: Some projects cost more but reduce carbon significantly. The platform must treat these "carbon-led" projects as equally valid — no negative visual treatment for projects with a higher cost than baseline. The value driver (financial, carbon, or both) determines how a project is judged.

Fuel Library

Alternative Fuel Modelling

A central registry of fuel types that the platform knows about — diesel, LPG, biomass, HVO, and so on. Each fuel carries an emission factor (how much CO₂ per unit) and an energy content value (how much energy per unit), which lets the platform calculate carbon impact and compare fuels on a like-for-like basis when modelling switches.

What we need to know about each fuel

Fuel name & category Default unit (litres, tonnes, etc.) Emission factor (kgCO₂e per unit) Energy content (kWh per unit) Factor source & year Default GHG scope Applicable compliance schemes

Fuel categories

Fossil liquid (diesel, petrol, kerosene) Fossil gas (natural gas, LPG) Biofuel (HVO, biodiesel, biogas) Solid (coal, biomass) Other
Extensible by design: Emission factors come pre-loaded from DEFRA, but admins can add custom fuel types and override factors where needed. The factor year is tracked so the platform can flag when data becomes stale.

Upload Template

Project Upload Template

A standardised spreadsheet template that lets consultants create projects in bulk instead of one at a time. The columns mirror the project and fuel data above — what the project is, what it measures, how much it saves. Some fields auto-populate (like carbon savings and emission factors) so users don't need to look them up manually.

Required information (must be filled in)

Project name Project type Technology / measure GHG Scope Baseline consumption Unit Projected consumption Annual saving

Optional / auto-calculated

Capital cost Payback period Carbon saving (auto) Fuel type Site / location Notes
Sequencing note: The template's columns are derived from the project data model. If the Scope 3 Foundation changes what a project looks like, the template will need updating too. The project model should be agreed first.

Audit Trail

Universal Admin Access

Every action taken by a True consultant is logged — who did it, which client account they were working in, what they changed, and when. This is essential for accountability in a multi-tenant environment where one consultant may work across dozens of client accounts.

What gets recorded

Who (named consultant + role) Where (client account) What (action taken) When (timestamp) How many records affected

How They Connect

The four data areas aren't independent — they share information and depend on each other in specific ways.

Project → Fuel Library

When a project involves fuel, the platform looks up the emission factor and default unit from the fuel library automatically. The fuel's GHG scope also pre-fills on the project.

Project → Upload Template

The template columns are a direct reflection of the project data model. Over half the template fields map straight to project fields — change one, change the other.

Fuel Library → Upload Template

When a user specifies a fuel type in the template, the platform auto-fills the unit and emission factor from the library — reducing manual input and error.

Audit Trail → Everything

Every create, update, or bulk upload across projects, fuels, and templates is logged. The audit trail sits underneath all three, recording every change.
