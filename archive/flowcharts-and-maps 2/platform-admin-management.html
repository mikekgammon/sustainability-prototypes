Foundation Phase · Requirement 01

Universal True Admin Access

Flowchart mapping the proposed access model — from consultant login through role assignment, account access, and audit logging. Replaces the current fragmented approach (impersonation, individual logins, mixed structures) with a single platform-level admin framework.

Current problems: Fragmented identity management, audit trail ambiguity, permission inconsistencies across accounts, operational inefficiencies, risk of modelling discrepancies. No standardised "True-level" parent access structure exists today.

Process Step
Decision Gate
1. Authentication & Role Assignment
Consultant opens TrueDI
Authentication
Single sign-on via True Admin credentials
One login per consultant. No client-specific credentials needed. SSO or standard auth — same entry point for all TrueCrew members regardless of which accounts they manage.
Identity
Platform resolves consultant identity
System identifies the individual consultant (e.g. "Otto Terrell — True_Admin"). All subsequent actions logged against this named identity, not a generic admin account.
Decision
Role assigned?
Check whether the consultant has been assigned the True_Admin global role by a platform administrator.
Yes — True_Admin
Access
Load assigned client accounts
Dashboard shows only the client accounts this consultant is assigned to. No visibility into unrelated accounts — data isolation maintained.
No — Not assigned
Blocked
Access denied
Consultant sees a message to contact their platform admin for role assignment. No fallback to impersonation or ad-hoc logins.
2. Account Navigation & Permissions
Consultant selects a client account
Decision
Consultant assigned to this account?
Account assignment logic: True_Admin users only see and can access accounts they've been explicitly assigned to. This is managed at platform admin level.
Yes — Assigned
Permissions
Full project access granted
Consultant can view all projects, create strategies and comparisons using any project within the account, manage modelling, and access reporting — without impersonating a client user.
Cross-user
Cross-user project modelling
True_Admin can build strategies that pull from any user's projects within the account. Removes the current limitation where strategies are siloed to individual user contexts.
No — Not assigned
Blocked
Account not visible
Account doesn't appear in the consultant's account list. No way to navigate to or access data for unassigned clients.
3. Audit Trail & Traceability
Consultant performs an action
Logging
Action logged with full identity context
Every action records: True_Admin role + individual consultant name + timestamp + client account + action type. No more ambiguity about who did what.
Audit record
Audit log entry structure
Actor: Otto Terrell (True_Admin)
Account: Acme Manufacturing
Action: Created strategy "Fleet Decarbonisation"
Timestamp: 2026-04-16 09:42:18
Projects referenced: 3 (cross-user)
Decision
Client-visible action?
Some actions are visible to the client (e.g. publishing a strategy). Others are internal only (e.g. reviewing a model).
Yes — Client-facing
Attribution
Show as "True Sustainability"
Client sees the action attributed to "True Sustainability" (or similar brand label) rather than an individual consultant name. Clean client experience.
No — Internal
Internal
Full consultant identity in logs
Internal audit log retains full consultant identity for compliance, debugging, and accountability.
4. Platform Admin — Role & Account Management
Platform admin opens admin panel
User management
View all True consultants
List of TrueCrew members with their current role (True_Admin or none) and assigned client accounts. Search and filter by name, role, or account.
Assign
Assign True_Admin role + client accounts
Admin selects a consultant, grants True_Admin role, and assigns specific client accounts. Granular — not all-or-nothing. Can be updated as account ownership changes.
Subtask
Ownership transfer
When a consultant leaves or moves accounts, admin can transfer ownership of projects and strategies to another True_Admin — no orphaned data.
Subtask
Bulk duplication & flexible copy
Admin can duplicate project structures, strategies, or templates across accounts. Enables standardised setup for new clients without rebuilding from scratch each time.
